# Data Model: AngleCraft MVP

**Feature**: [spec.md](./spec.md) | [plan.md](./plan.md) | [research.md](./research.md)

## Overview

All data is stored in Supabase (PostgreSQL). There is no Supabase Auth —
sessions are anonymous, identified by a UUID token stored in an HTTP-only
cookie. All schema changes are managed via **Supabase Migrations** in
`supabase/migrations/`. Generated images are stored in **Supabase Storage**
(private bucket `ad-creatives`); the database stores storage paths, not base64
blobs. Image generation jobs are managed via **Supabase Queue** (`pgmq`).

All data access happens server-side: Next.js route handlers are the
orchestration layer — they handle all database reads/writes, session
management, queue operations, and Storage signed URLs. Supabase Edge
Functions are AI-only — they call OpenAI and return structured results without
writing to the database (except `process-image-queue` which is cron-triggered
and self-contained).

## Entities

### Session

The root entity. An anonymous browser-bound workspace that owns all product
input and generated artifacts.

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `id` | UUID | PK, `gen_random_uuid()` | Internal identifier |
| `token` | UUID | UNIQUE, NOT NULL | Cookie token (UUID v4) |
| `status` | `session_status` | NOT NULL, default `'input'` | Current pipeline state |
| `created_at` | timestamptz | NOT NULL, default `now()` | Creation timestamp |
| `expires_at` | timestamptz | NOT NULL, `created_at + interval '7 days'` | Expiry timestamp |

**State transitions** (`session_status` enum):

```text
input → extracting → analyzing → angles_generated → paid → generating → complete
```

Any state can transition to `failed` (with a retry returning to the previous
state). The `paid` transition is triggered by Stripe webhook confirmation.
The `generating` state covers the post-payment status flow (concepts →
creatives → testing plan).

**Relationships**: Has one ProductInput, one BuyerInsights, five AdAngles,
three AdCreatives (after payment), one TestingPlan (after payment), at most
one Payment.

### ProductInput

The user-submitted product reference and derived context.

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `id` | UUID | PK | Internal identifier |
| `session_id` | UUID | FK → sessions.id, NOT NULL | Owning session |
| `input_type` | `input_type` | NOT NULL | `'url'` or `'photo'` |
| `url` | text | NULL (required if `input_type = 'url'`) | Submitted product URL |
| `image_storage_path` | text | NULL (required if `input_type = 'photo'`) | Supabase Storage path for uploaded photo |
| `extracted_name` | text | NULL | Product name from extraction |
| `extracted_description` | text | NULL | Product description |
| `extracted_price` | text | NULL | Price (as text — varies by currency format) |
| `extracted_features` | text[] | NULL | Bullet point features |
| `extracted_image_url` | text | NULL | Primary product image URL (from URL extraction) |
| `product_context` | jsonb | NULL | Full structured context (category, benefits, audience signals) — generated by Edge Function |
| `created_at` | timestamptz | NOT NULL, default `now()` | Submission timestamp |

**Validation rules**:
- Exactly one of `url` or `image_storage_path` must be non-null based on `input_type`.
- `url` must be a valid HTTP/HTTPS URL.
- Uploaded photos are stored in Supabase Storage (`product-photos` bucket), not as base64 in the database.

### AdAngle

One of five priority creative framings. Generated by the `generate-angles` Edge
Function. The AI writes hooks and rationales for a fixed pool of 10 candidate
labels — **scoring and top-5 selection are deterministic and applied in code**, not
by the AI.

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `id` | UUID | PK | Internal identifier |
| `session_id` | UUID | FK → sessions.id, NOT NULL | Owning session |
| `angle_label` | text | NOT NULL | One of 10 candidate taxonomy values (see below) |
| `hook` | text | NOT NULL | Single attention-grabbing opening line |
| `rationale` | text | NULL | AI explanation of why this angle works for this product |
| `score` | numeric(5,2) | NULL | Deterministic score (1-10 scale) for priority ranking and top-3 selection |
| `is_selected` | boolean | NOT NULL, default `false` | True if in top 3 selected for paid creatives |
| `created_at` | timestamptz | NOT NULL, default `now()` | Generation timestamp |

**Angle taxonomy (10 candidate labels):** `pain_point`, `convenience`,
`time_saving`, `gift`, `lifestyle`, `emotional`, `educational`, `aspiration`,
`transformation`, `social_proof`.

**Scoring:** Base scores per taxonomy angle live in `BASE_SCORES` in
`supabase/functions/generate-angles/index.ts` across four criteria
(`purchaseIntent`, `audienceReach`, `creativePotential`, `emotionalStrength`,
each 1-10). The final score is
`purchaseIntent*0.40 + audienceReach*0.25 + creativePotential*0.20 + emotionalStrength*0.15`.
Product-category boosts from `CATEGORY_BOOSTS` are applied deterministically,
then the top 5 angles become Priority #1–#5. The `score` column is
`numeric(5,2)` (not integer) to store decimal values.

**Validation rules**:
- Exactly five AdAngles per session, each with a distinct `angle_label` (the top 5 priorities from the 10 candidate pool).
- `angle_label` must be one of the 10 candidate taxonomy values.
- `is_selected = true` for exactly three of the five (set after deterministic scoring).

### AdCreative

A ready-to-run ad creative for a selected ad angle (paid output). Copy is
generated by the `generate-copy` Edge Function; image is generated
asynchronously via the `process-image-queue` Edge Function and stored in
Supabase Storage.

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `id` | UUID | PK | Internal identifier |
| `session_id` | UUID | FK → sessions.id, NOT NULL | Owning session |
| `angle_id` | UUID | FK → ad_angles.id, NOT NULL | Parent ad angle (must be selected) |
| `creative_index` | int2 | NOT NULL, default `0` | Stable 1–3 order for the selected creatives (preserves priority across batch inserts) |
| `concept` | text | NOT NULL | Ad concept description (generated by `generate-concepts` Edge Function) |
| `headline` | text | NULL | Ad headline (populated by `generate-copy` Edge Function) |
| `primary_text` | text | NULL | Body copy (populated by `generate-copy` Edge Function) |
| `cta` | text | NULL | Call-to-action (populated by `generate-copy` Edge Function) |
| `image_text` | text | NULL | Optional text overlay baked into the generated image (e.g. for feed creatives) |
| `image_storage_path` | text | NULL | Supabase Storage path for generated ad image (populated by `process-image-queue` Edge Function) |
| `image_status` | `image_status` | NOT NULL, default `'pending'` | Image generation status |
| `created_at` | timestamptz | NOT NULL, default `now()` | Generation timestamp |

**Validation rules**:
- Exactly three AdCreatives per session (one per selected angle).
- `angle_id` must reference an AdAngle where `is_selected = true`.
- `creative_index` must be 1, 2, or 3 and unique per session.
- `image_storage_path` is populated when `image_status = 'complete'`.
- `created_at` cannot be used for ordering because the three creatives are inserted in a single batch and share the same timestamp; always use `creative_index` for ordering.

### TestingPlan

Structured testing recommendation (paid output). Generated by the
`generate-testing-plan` Edge Function. The playbook is Meta Ads only (no
TikTok). A master `campaignStrategy` object is built deterministically in code
and every section is derived from it; deterministic fields are force-overwritten
after the AI returns so no section can independently deviate.

The `actionPlan` field was removed from the rendered playbook because its
contents (platform, campaign type, budget, metrics, audience strategy, decision
rule) were duplicated inside `recommendedFirstTest` and `testingPlan`. The data
field is still accepted for backward compatibility but is no longer shown in the
UI or PDF.

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `id` | UUID | PK | Internal identifier |
| `session_id` | UUID | FK → sessions.id, UNIQUE, NOT NULL | Owning session (1:1) |
| `plan_content` | jsonb | NOT NULL | Structured playbook (campaign strategy, testing phases, budgets, per-creative guidance, workflow, disclaimer) |
| `created_at` | timestamptz | NOT NULL, default `now()` | Generation timestamp |

### Payment

Record of a one-time Stripe charge against a session.

| Field | Type | Constraints | Description |
|-------|------|-------------|-------------|
| `id` | UUID | PK | Internal identifier |
| `session_id` | UUID | FK → sessions.id, UNIQUE, NOT NULL | Owning session (1:1) |
| `stripe_session_id` | text | UNIQUE, NOT NULL | Stripe Checkout Session ID |
| `status` | `payment_status` | NOT NULL | `'pending'`, `'succeeded'`, `'failed'` |
| `amount` | integer | NOT NULL | Amount in cents (900 = $9.00) |
| `currency` | text | NOT NULL, default `'usd'` | ISO currency code |
| `created_at` | timestamptz | NOT NULL, default `now()` | Creation timestamp |
| `completed_at` | timestamptz | NULL | When payment succeeded/failed |

**Validation rules**:
- At most one Payment per session with `status = 'succeeded'` (prevents duplicate charges).
- `amount` is always 900 (cents) for the MVP.

## SQL Schema (Supabase Migration)

This is the content of the initial migration file
`supabase/migrations/<timestamp>_init.sql`. It is applied via
`supabase migration up` (local) or `supabase db push` (remote).

```sql
-- Enable extensions
create extension if not exists pgmq;
create extension if not exists pg_cron;

-- Enums
create type session_status as enum (
  'input', 'extracting', 'analyzing', 'angles_generated',
  'paid', 'generating', 'complete', 'failed'
);
create type input_type as enum ('url', 'photo');
create type payment_status as enum ('pending', 'succeeded', 'failed');
create type image_status as enum ('pending', 'processing', 'complete', 'failed');

-- Sessions
create table sessions (
  id uuid primary key default gen_random_uuid(),
  token uuid unique not null default gen_random_uuid(),
  status session_status not null default 'input',
  created_at timestamptz not null default now(),
  expires_at timestamptz not null default (now() + interval '7 days')
);
create index idx_sessions_token on sessions(token);

-- Product Inputs
create table product_inputs (
  id uuid primary key default gen_random_uuid(),
  session_id uuid not null references sessions(id) on delete cascade,
  input_type input_type not null,
  url text,
  image_storage_path text,
  extracted_name text,
  extracted_description text,
  extracted_price text,
  extracted_features text[],
  extracted_image_url text,
  product_context jsonb,
  created_at timestamptz not null default now()
);

-- Buyer Insights
create table buyer_insights (
  id uuid primary key default gen_random_uuid(),
  session_id uuid not null unique references sessions(id) on delete cascade,
  buyer_profile text not null,
  main_desire text not null,
  pain_points text[] not null,
  buying_triggers text[] not null,
  objections text[] not null,
  created_at timestamptz not null default now()
);

-- Ad Angles
create table ad_angles (
  id uuid primary key default gen_random_uuid(),
  session_id uuid not null references sessions(id) on delete cascade,
  angle_label text not null,
  hook text not null,
  rationale text,
  score numeric(5,2),
  is_selected boolean not null default false,
  created_at timestamptz not null default now()
);

-- Ad Creatives
create table ad_creatives (
  id uuid primary key default gen_random_uuid(),
  session_id uuid not null references sessions(id) on delete cascade,
  angle_id uuid not null references ad_angles(id) on delete cascade,
  concept text not null,
  headline text,
  primary_text text,
  cta text,
  image_text text,
  image_storage_path text,
  image_status image_status not null default 'pending',
  created_at timestamptz not null default now()
);

-- Testing Plans
create table testing_plans (
  id uuid primary key default gen_random_uuid(),
  session_id uuid not null unique references sessions(id) on delete cascade,
  plan_content jsonb not null,
  created_at timestamptz not null default now()
);

-- Payments
create table payments (
  id uuid primary key default gen_random_uuid(),
  session_id uuid not null unique references sessions(id) on delete cascade,
  stripe_session_id text unique not null,
  status payment_status not null,
  amount integer not null,
  currency text not null default 'usd',
  created_at timestamptz not null default now(),
  completed_at timestamptz
);
create index idx_payments_status on payments(status);

-- Create image generation queue
select pgmq.create('image_generation');

-- Schedule queue processor (every 30 seconds)
select cron.schedule(
  'process-image-queue',
  '*/30 seconds * * * *',
  $$select net.http_post(
    url := 'https://<project-ref>.supabase.co/functions/v1/process-image-queue',
    headers := '{"Content-Type": "application/json"}'::jsonb
  )$$
);
```

**Note**: The `<project-ref>` placeholder in the cron schedule must be
replaced with the actual Supabase project reference during deployment.

## Session Lifecycle

```text
1. User submits URL/photo
   → Next.js route handler creates session (status: 'input')
   → Set session_token cookie
   → If photo: upload to Supabase Storage (product-photos bucket)
   → Transition to 'extracting'
   → Extract product info with cheerio (Next.js route handler)
   → Write extracted data to product_inputs

2. Analyze product
   → Next.js route handler invokes analyze-product Edge Function (AI-only)
   → Edge Function returns product context + Buyer Insights
   → Next.js route handler writes buyer_insights + product_context to DB
   → Transition to 'analyzing'

3. Generate ad angles
   → Next.js route handler invokes generate-angles Edge Function (AI-only)
   → Edge Function returns 10 candidate angles + hooks + rationale (AI uses fixed 10-label pool; NO scores from AI)
   → Edge Function applies deterministic base scores + product-category boosts in code, then selects top 5 priorities
   → Next.js route handler writes 5 ad_angles rows (the top 5 priorities)
   → Next.js route handler selects top 3 by score, marks is_selected = true
   → Status: 'angles_generated' (free preview ready)

4. User pays via Stripe
   → Next.js route handler creates payment record (status: 'pending')
   → Redirect to Stripe Checkout
   → Webhook confirms → Next.js route handler updates payment status: 'succeeded'
   → Session status: 'paid'

5. Post-payment generation
   → Status: 'generating'
   → Next.js route handler invokes generate-concepts Edge Function (AI-only)
   → Edge Function returns 3 concepts
   → Next.js route handler creates 3 ad_creatives rows with concept text
   → Next.js route handler invokes generate-copy Edge Function (AI-only)
   → Edge Function returns headlines, primary text, CTA
   → Next.js route handler updates ad_creatives rows with copy
   → Next.js route handler enqueues 3 image generation jobs to queue
   → process-image-queue Edge Function (cron-triggered, self-contained):
     → Reads queue, generates image with OpenAI gpt-image-1
     → Uploads to Supabase Storage (ad-creatives bucket)
     → Updates ad_creatives.image_storage_path + image_status
     → Deletes queue message
   → Next.js route handler invokes generate-testing-plan Edge Function (AI-only)
   → Edge Function returns testing plan
   → Next.js route handler creates testing_plans row, updates session status
   → Status: 'complete' (when all images + plan ready)

Error recovery:
   → Any step can fail → status: 'failed'
   → Retry returns to the step that failed
   → Session data and payment preserved
   → Queue jobs retry automatically (max 3, then archive)
```

## Data Access Patterns

All data access is server-side. **Next.js route handlers** are the
orchestration layer — they handle all database reads/writes, session
management, queue operations, and Storage signed URLs. **Supabase Edge
Functions** are AI-only — they call OpenAI and return structured results;
they do not write to the database or access Storage (except
`process-image-queue` which is cron-triggered and self-contained).

| Operation | Layer | Table(s) / Service |
|-----------|-------|--------------------|
| Create session | Next.js route handler | sessions, product_inputs |
| Upload product photo | Next.js route handler | Supabase Storage (product-photos) |
| Extract product info | Next.js route handler (cheerio) | product_inputs (update) |
| Analyze product (AI) | Edge Function (`analyze-product`) | — (returns results only) |
| Store Buyer Insights | Next.js route handler | buyer_insights, product_inputs |
| Generate angles (AI) | Edge Function (`generate-angles`) | — (returns results only) |
| Store angles + select top 3 | Next.js route handler | ad_angles (insert + update is_selected) |
| Create payment | Next.js route handler | payments |
| Confirm payment | Next.js route handler (webhook) | payments, sessions |
| Generate concepts (AI) | Edge Function (`generate-concepts`) | — (returns results only) |
| Store concepts | Next.js route handler | ad_creatives (insert) |
| Generate copy (AI) | Edge Function (`generate-copy`) | — (returns results only) |
| Store copy | Next.js route handler | ad_creatives (update) |
| Enqueue image jobs | Next.js route handler | pgmq (image_generation queue) |
| Generate images (AI) + store | Edge Function (`process-image-queue`) | ad_creatives (update), Storage |
| Generate testing plan (AI) | Edge Function (`generate-testing-plan`) | — (returns results only) |
| Store testing plan | Next.js route handler | testing_plans, sessions (update status) |
| Get all results | Next.js Server Component | all tables by session_id |
| Get signed image URLs | Next.js Server Component | Supabase Storage |
| Generate PDF | Next.js route handler | all tables + Storage |
